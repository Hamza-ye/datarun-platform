# Walk-Throughs

Domain-specific real-world examples that show how concrete operational situations map to the platform's scenarios and architectural primitives.

These are **not scenarios** — they name real things (ITNs, villages, households) and describe domain-specific flows. They exist to:

1. Validate that scenarios capture the right operational patterns
2. Show how architectural primitives compose to support real work
3. Serve as reference material for implementation spikes and acceptance testing

Each walk-through references the scenario(s) it exercises and notes any architectural pressure points encountered.
